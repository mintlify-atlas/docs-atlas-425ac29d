## Prerequisites

Before you begin, ensure you have the following installed:

- **.NET 10 SDK** - [Download here](https://dotnet.microsoft.com/download)
- **Node.js** (v18 or higher) - [Download here](https://nodejs.org/)
- **PostgreSQL** (v14 or higher) - [Download here](https://www.postgresql.org/download/)
- **Git** - [Download here](https://git-scm.com/downloads)

<Note>
The backend previously supported SQL Server but has been migrated to PostgreSQL for better deployment compatibility on platforms like Railway.
</Note>

## Installation

<Steps>
  <Step title="Clone the repository">
    Get the source code from your repository:

    ```bash
    git clone <your-repository-url>
    cd legacy-migration-task-manager
    ```
  </Step>

  <Step title="Set up the database">
    Create a PostgreSQL database for the application:

    ```bash
    # Connect to PostgreSQL
    psql -U postgres

    # Create the database
    CREATE DATABASE project_template_scharp;

    # Exit psql
    \q
    ```

    <Info>
    The application will automatically create tables using Entity Framework migrations when it first runs.
    </Info>
  </Step>

  <Step title="Configure the backend">
    Set up your database connection string. You can do this in two ways:

    **Option 1: Environment variable (recommended)**

    ```bash
    # Linux/macOS
    export DATABASE_URL="Host=localhost;Database=project_template_scharp;Username=postgres;Password=your_password"

    # Windows (PowerShell)
    $env:DATABASE_URL="Host=localhost;Database=project_template_scharp;Username=postgres;Password=your_password"
    ```

    **Option 2: Create appsettings.json**

    Create `source/backend/appsettings.json`:

    ```json
    {
      "ConnectionStrings": {
        "ProjectTemplateConnection": "Host=localhost;Database=project_template_scharp;Username=postgres;Password=your_password"
      },
      "Jwt": {
        "Key": "YourSecretKeyHere_MinimumLength32Characters!",
        "Issuer": "TemplateAPI",
        "Audience": "TemplateAPI"
      },
      "Logging": {
        "LogLevel": {
          "Default": "Information",
          "Microsoft.AspNetCore": "Warning"
        }
      },
      "AllowedHosts": "*"
    }
    ```

    <Note>
    The application checks for connection strings in this order:
    1. `ConnectionStrings:ProjectTemplateConnection` in appsettings.json
    2. `DATABASE_URL` in configuration
    3. `DATABASE_URL` environment variable
    </Note>
  </Step>

  <Step title="Install backend dependencies">
    Navigate to the backend directory and restore NuGet packages:

    ```bash
    cd source/backend
    dotnet restore
    ```
  </Step>

  <Step title="Apply database migrations">
    Initialize the database schema with Entity Framework:

    ```bash
    dotnet ef database update
    ```

    This creates all necessary tables, including:
    - `tbl_users` - User accounts
    - `tbl_tasks` - Task records
    - `tbl_projects` - Project organization
    - `tbl_states` - Task states (Pending, In Progress, Completed)
    - `tbl_priorities` - Priority levels
    - `tbl_comments` - Task comments
    - `tbl_history` - Audit trail
    - `tbl_notifications` - User notifications
  </Step>

  <Step title="Start the backend server">
    Run the .NET API server:

    ```bash
    dotnet run
    ```

    Or with hot reload for development:

    ```bash
    dotnet watch run
    ```

    The backend will start at `https://localhost:7067`

    <Info>
    Access the OpenAPI documentation at: `https://localhost:7067/swagger/index.html` (development mode only)
    </Info>
  </Step>

  <Step title="Install frontend dependencies">
    Open a new terminal and navigate to the frontend directory:

    ```bash
    cd source
    npm install
    ```

    This installs all required packages including:
    - React 19
    - React Router DOM
    - Tailwind CSS 4
    - Webpack and build tools
  </Step>

  <Step title="Start the frontend">
    Start the React development server:

    ```bash
    npm run dev
    ```

    Or just start without opening browser:

    ```bash
    npm start
    ```

    The frontend will start at `http://localhost:8080`
  </Step>

  <Step title="Access the application">
    Open your browser and navigate to:

    ```
    http://localhost:8080
    ```

    You'll be redirected to the login page.
  </Step>
</Steps>

## First login

<Note>
You'll need to create a user account in the database first, or use seed data if available. The application uses password comparison without hashing for this educational project.
</Note>

To create a test user, connect to your database and run:

```sql
INSERT INTO tbl_users (username, password, active, usercreate, creation_date)
VALUES ('admin', 'admin123', true, 1, NOW());
```

Then log in with:
- **Username**: `admin`
- **Password**: `admin123`

<Info>
Upon successful login, you'll receive a JWT token that's automatically stored and used for subsequent API requests.
</Info>

## Verify the installation

After logging in, you should see the main task manager interface with:

1. **Navigation sidebar** - Access different modules (Tasks, Projects, Comments, etc.)
2. **Task list** - View and manage tasks
3. **Statistics** - See task counts and status breakdowns

## Development workflow

### Backend development

The backend uses hot reload with `dotnet watch run`. Any changes to C# files will automatically rebuild and restart the server.

```bash
cd source/backend
dotnet watch run
```

### Frontend development

Webpack dev server provides hot module replacement:

```bash
cd source
npm run dev
```

### Building for production

<CodeGroup>
```bash Backend
cd source/backend
dotnet publish -c Release
```

```bash Frontend
cd source
npm run build
```
</CodeGroup>

The frontend build outputs to `dist/` and can be copied to the backend's `wwwroot/` folder for unified deployment.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Database connection failed">
    **Error**: `Database connection string is not configured`

    **Solution**: Ensure you've set the `DATABASE_URL` environment variable or created `appsettings.json` with the correct connection string.

    Verify PostgreSQL is running:
    ```bash
    # Check PostgreSQL status
    sudo systemctl status postgresql  # Linux
    brew services list                # macOS
    ```
  </Accordion>

  <Accordion title="Port already in use">
    **Error**: `Address already in use`

    **Solution**: Another process is using port 7067 (backend) or 8080 (frontend).

    Find and kill the process:
    ```bash
    # Find process using port
    lsof -i :7067  # macOS/Linux
    netstat -ano | findstr :7067  # Windows

    # Kill the process
    kill -9 <PID>
    ```
  </Accordion>

  <Accordion title="CORS errors in browser">
    **Error**: `Access to fetch at 'https://localhost:7067' has been blocked by CORS policy`

    **Solution**: Ensure the backend CORS policy is configured correctly. Check `Program.cs:15-23` - the `AllowFrontend` policy should allow your frontend origin.

    The current configuration allows all origins in development.
  </Accordion>

  <Accordion title="JWT authentication failing">
    **Error**: `Unauthorized` responses from API

    **Solution**: 
    1. Ensure you've logged in and received a token
    2. Check that the token is being sent in the `Authorization` header
    3. Verify the JWT secret key is at least 32 characters
    4. Check token expiration settings
  </Accordion>
</AccordionGroup>
