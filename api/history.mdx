## Get History By Task ID

<Card title="GET /api/History/task/{taskId}" icon="clock-rotate-left">
  Retrieve the complete history of changes for a specific task
</Card>

### Path Parameters

<ParamField path="taskId" type="integer" required>
  The task ID to retrieve history for
</ParamField>

### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
</ParamField>

### Response

<ResponseField name="success" type="boolean">
  Indicates if the request was successful
</ResponseField>

<ResponseField name="data" type="array">
  Array of history records
  <Expandable title="History object">
    <ResponseField name="Id" type="integer">
      History record ID
    </ResponseField>
    <ResponseField name="TaskId" type="integer">
      Associated task ID
    </ResponseField>
    <ResponseField name="Action" type="string">
      The action performed (e.g., "Created", "Updated", "Status Changed")
    </ResponseField>
    <ResponseField name="Description" type="string">
      Detailed description of the change
    </ResponseField>
    <ResponseField name="CreationDate" type="string">
      When the action occurred (ISO 8601 format)
    </ResponseField>
    <ResponseField name="Active" type="boolean">
      Whether the history record is active
    </ResponseField>
  </Expandable>
</ResponseField>

### Example Request

```bash
curl -X GET https://api.example.com/api/History/task/1 \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

```json 200 OK
      "Id": 2,
      "TaskId": 1,
      "Action": "Status Changed",
      "Description": "Status changed from 'To Do' to 'In Progress'",
      "CreationDate": "2026-03-02T09:15:00Z",
      "Active": true
    },
    {
      "Id": 3,
      "TaskId": 1,
      "Action": "Updated",
      "Description": "Estimated hours changed from 40 to 50",
      "CreationDate": "2026-03-04T15:30:00Z",
```

```json 400 Bad Request
{
  "success": false,
  "message": "Task not found or invalid task ID"
```

## Create History Record

<Card title="POST /api/History" icon="plus">
  Manually create a history record for a task
</Card>

### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
</ParamField>

### Request Body

<ParamField body="TaskId" type="integer" required>
  The task ID to associate the history record with
</ParamField>

<ParamField body="Action" type="string" required>
  The action being recorded (e.g., "Comment Added", "File Attached", "Assigned")
</ParamField>

<ParamField body="Description" type="string" required>
  Detailed description of the action or change
</ParamField>

### Example Request

```bash
curl -X POST https://api.example.com/api/History \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "TaskId": 1,
    "Action": "Comment Added",
    "Description": "User John Doe added a comment about migration progress"
  }'
```

### Example Response

<CodeGroup>
```json 200 OK
{
  "success": true,
  "data": {
    "Id": 6,
    "TaskId": 1,
    "Action": "Comment Added",
    "Description": "User John Doe added a comment about migration progress",
    "CreationDate": "2026-03-04T16:45:00Z",
    "Active": true
  }
}
```

```json 400 Bad Request
{
  "success": false,
  "message": "Invalid request data"
}
```
</CodeGroup>

### Status Codes

- `200 OK` - History record successfully created
- `400 Bad Request` - Missing required fields or invalid data format

## Delete History Record

<Card title="DELETE /api/History/{historyId}" icon="trash">
  Delete a history record (soft delete - marks as inactive)
</Card>

### Path Parameters

<ParamField path="historyId" type="integer" required>
  The history record ID to delete
</ParamField>

### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
</ParamField>

### Example Request

```bash
curl -X DELETE https://api.example.com/api/History/1 \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

### Example Response

<CodeGroup>
```json 200 OK
{
  "success": true,
  "message": "History deleted successfully"
}
```

```json 404 Not Found
{
  "success": false,
  "message": "History not found"
}
```

```json 400 Bad Request
{
  "success": false,
  "message": "Error deleting history: Database error"
}
```
</CodeGroup>

### Status Codes

- `200 OK` - History record successfully deleted
- `404 Not Found` - History record with specified ID doesn't exist
- `400 Bad Request` - Server error occurred during deletion

<Warning>
  This endpoint performs a soft delete by marking the history record as inactive (`Active = false`). The history data remains in the database for audit purposes.
</Warning>

## Common Error Responses

All history endpoints may return the following error responses:

### 400 Bad Request

```json
{
  "success": false,
  "message": "Error description"
}
```

This error occurs when:
- Required fields are missing (Action, Description, TaskId)
- Request data is malformed or invalid
- Database operation fails
- Invalid history ID provided

### 404 Not Found

```json
{
  "success": false,
  "message": "History not found"
}
```

This error occurs when:
- The specified history ID doesn't exist
- The history record has been deleted

## Response Format

<Note>
  The History API uses a slightly different response format compared to other endpoints (lowercase "success" and no "Success"/"Message" wrapper in GET responses).
</Note>

## Usage Examples

### Track Task History in JavaScript

```javascript
const getTaskHistory = async (taskId, token) => {
  const response = await fetch(
    `https://api.example.com/api/History/task/${taskId}`,
    {
      headers: {
        'Authorization': `Bearer ${token}`
      }
    }
  );
  
  const data = await response.json();
  
  if (data.success) {
    return data.data.sort((a, b) => 
      new Date(b.CreationDate) - new Date(a.CreationDate)
    );
  }
  
  return [];
};
```

### Create History Entry

```javascript
const logTaskAction = async (taskId, action, description, token) => {
  const response = await fetch('https://api.example.com/api/History', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${token}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      TaskId: taskId,
      Action: action,
      Description: description
    })
  });
  
  return await response.json();
};

// Usage
await logTaskAction(
  1, 
  'Milestone Reached', 
  'Completed phase 1 of database migration',
  userToken
);
```

## Best Practices

<Tip>
  **Audit Trail Best Practices:**
  
  - Automatically create history records when tasks are created, updated, or deleted
  - Include the username or user ID in the description for accountability
  - Use consistent action names across your application
  - Store enough detail in the description to reconstruct what changed
  - Display history in reverse chronological order (newest first)
  - Consider pagination for tasks with extensive history
</Tip>

<Warning>
  **Considerations:**
  
  - History records should typically be created automatically by the system, not manually
  - The current API allows manual history creation, which could lead to inconsistent audit trails
  - Consider implementing automatic history tracking at the service layer
  - Avoid deleting history records unless absolutely necessary for compliance reasons
</Warning>
