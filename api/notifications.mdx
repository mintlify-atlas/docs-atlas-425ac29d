## Get Notifications By User ID

<Card title="GET /api/Notifications/user/{userId}" icon="bell">
  Retrieve all notifications for a specific user
</Card>

### Path Parameters

<ParamField path="userId" type="integer" required>
  The user ID to retrieve notifications for
</ParamField>

### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
</ParamField>

### Response

<ResponseField name="Success" type="boolean">
  Indicates if the request was successful
</ResponseField>

<ResponseField name="Message" type="string">
  Response message
</ResponseField>

<ResponseField name="Data" type="array">
  Array of notification objects
  <Expandable title="Notification object">
    <ResponseField name="Id" type="integer">
      Notification ID
    </ResponseField>
    <ResponseField name="UserId" type="integer">
      User ID who receives the notification
    </ResponseField>
    <ResponseField name="UserName" type="string">
      Username of the recipient
    </ResponseField>
    <ResponseField name="TaskId" type="integer">
      Associated task ID
    </ResponseField>
    <ResponseField name="TaskTitle" type="string">
      Title of the associated task
    </ResponseField>
    <ResponseField name="Title" type="string">
      Notification title
    </ResponseField>
    <ResponseField name="Message" type="string">
      Notification message content
    </ResponseField>
    <ResponseField name="Type" type="string">
      Notification type (e.g., "Task Assigned", "Task Updated", "Comment Added")
    </ResponseField>
    <ResponseField name="Read" type="boolean">
      Whether the notification has been read
    </ResponseField>
    <ResponseField name="CreationDate" type="string">
      When the notification was created (ISO 8601 format)
    </ResponseField>
    <ResponseField name="ModificationDate" type="string">
      When the notification was last modified (ISO 8601 format)
    </ResponseField>
    <ResponseField name="Active" type="boolean">
      Whether the notification is active
    </ResponseField>
  </Expandable>
</ResponseField>


## Create Notification

<Card title="POST /api/Notifications" icon="plus">
  Create a new notification for a user
</Card>

### Headers

<ParamField header="Authorization" type="string" required>
  Bearer token for authentication
</ParamField>

### Request Body

<ParamField body="UserId" type="integer" required>
  The user ID to send the notification to
</ParamField>

<ParamField body="TaskId" type="integer" required>
  The associated task ID
</ParamField>

<ParamField body="Title" type="string" required>
  Notification title
</ParamField>

<ParamField body="Message" type="string" required>
  Notification message content
</ParamField>

<ParamField body="Type" type="string" required>
  Notification type (e.g., "Task Assigned", "Task Updated", "Comment Added", "Deadline Approaching")
</ParamField>

### Example Request

```bash
curl -X POST https://api.example.com/api/Notifications \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "UserId": 5,
    "TaskId": 1,
    "Title": "Task Due Soon",
    "Message": "The task 'Migrate database schema' is due in 2 days",
    "Type": "Deadline Approaching"
  }'
```

### Example Response

<CodeGroup>
```json 201 Created
{
  "Success": true,
  "Message": "Notification created successfully",
  "Data": {
    "Id": 3,
    "UserId": 5,
    "UserName": "John Doe",
    "TaskId": 1,
    "TaskTitle": "Migrate database schema",
    "Title": "Task Due Soon",
    "Message": "The task 'Migrate database schema' is due in 2 days",
    "Type": "Deadline Approaching",
    "Read": false,
    "CreationDate": "2026-03-04T16:00:00Z",
    "ModificationDate": null,
    "Active": true
  }
}
```

```json 400 Bad Request
{
  "Success": false,
  "Message": "Invalid notification data",
  "Data": null
}
```
</CodeGroup>

### Status Codes

- `201 Created` - Notification successfully created
- `400 Bad Request` - Invalid request data or validation error
- `401 Unauthorized` - Missing or invalid authentication token

## Usage Examples

### Load User Notifications

```javascript
const loadNotifications = async (userId, token) => {
  const response = await fetch(
    `https://api.example.com/api/Notifications/user/${userId}`,
    {
      headers: {
        'Authorization': `Bearer ${token}`
      }
    }
  );
  
  const data = await response.json();
  
  if (data.Success) {
    // Separate unread and read notifications
    const unread = data.Data.filter(n => !n.Read);
    const read = data.Data.filter(n => n.Read);
    
    return { unread, read, all: data.Data };
  }
  
  return { unread: [], read: [], all: [] };
};
```

### Get Unread Count for Badge

```javascript
const getUnreadCount = async (userId, token) => {
  const response = await fetch(
    `https://api.example.com/api/Notifications/user/${userId}/unread-count`,
    {
      headers: {
        'Authorization': `Bearer ${token}`
      }
    }
  );
  
  const data = await response.json();
  return data.Success ? data.Data : 0;
};

// Update badge in UI
const count = await getUnreadCount(currentUserId, authToken);
document.getElementById('notification-badge').textContent = count;
```

### Mark Notification as Read

```javascript
const markAsRead = async (notificationId, token) => {
  const response = await fetch(
    `https://api.example.com/api/Notifications/mark-read/${notificationId}`,
    {
      method: 'PUT',
      headers: {
        'Authorization': `Bearer ${token}`
      }
    }
  );
  
  return await response.json();
};

// Usage: Mark notification as read when user clicks it
notificationElement.addEventListener('click', async () => {
  await markAsRead(notification.Id, authToken);
  // Refresh notification list or update UI
});
```

### Create Notification

```javascript
const sendNotification = async (userId, taskId, title, message, type, token) => {
  const response = await fetch('https://api.example.com/api/Notifications', {
    method: 'POST',
    headers: {
      'Authorization': `Bearer ${token}`,
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      UserId: userId,
      TaskId: taskId,
      Title: title,
      Message: message,
      Type: type
    })
  });
  
  return await response.json();
};

// Usage: Notify user when assigned to task
await sendNotification(
  5,
  1,
  'New Task Assignment',
  'You have been assigned to "Migrate database schema"',
  'Task Assigned',
  authToken
);
```

## Best Practices

<Tip>
  **Notification Best Practices:**
  
  - Poll for unread count periodically to update notification badges
  - Use WebSockets or SignalR for real-time notification delivery in production
  - Group notifications by type or task to reduce notification fatigue
  - Implement notification preferences so users can control what they receive
  - Automatically mark notifications as read when users view the associated task
  - Archive or delete old notifications to keep the notification list manageable
  - Use descriptive titles and messages that clearly indicate the action required
</Tip>

<Warning>
  **Considerations:**
  
  - There is no endpoint to delete notifications - consider implementing this
  - Consider implementing notification preferences/settings per user
  - Add pagination for users with many notifications
  - Implement email or push notification integration for important alerts
  - Consider rate limiting notification creation to prevent spam
</Warning>
