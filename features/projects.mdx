## Overview

Projects provide a way to organize and group related tasks. Each task must be associated with a project, making projects a fundamental organizational unit in the system.

## Project Data Structure

The `TblProjects` model represents a project:

<ParamField path="id" type="integer">
  Unique project identifier (auto-generated)
</ParamField>

<ParamField path="name" type="string" required>
  Project name (required)
</ParamField>

<ParamField path="description" type="string">
  Project description (optional)
</ParamField>

<ParamField path="usercreate" type="integer">
  ID of user who created the project
</ParamField>

<ParamField path="usermod" type="integer">
  ID of user who last modified the project
</ParamField>

<ParamField path="creationDate" type="DateTime">
  Timestamp when project was created
</ParamField>

<ParamField path="modificationDate" type="DateTime">
  Timestamp of last modification
</ParamField>

<ParamField path="active" type="boolean">
  Whether the project is active (soft delete flag)
</ParamField>

## API Endpoints

### Get All Projects

Retrieves all active projects in the system.

```http
GET /api/projects
Authorization: Bearer {token}
```

<CodeGroup>
```javascript React Frontend

  const response = await fetch('/api/projects', {
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${token}`
    }

  const result = await response.json();
  if (result.success) {
```

### Create Project

Creates a new project.

```http
POST /api/projects
Authorization: Bearer {token}
Content-Type: application/json
```

<CodeGroup>
```javascript React Frontend
const handleSubmitAdd = async () => {
  const response = await fetch('/api/projects', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Authorization': `Bearer ${token}`
    },
    body: JSON.stringify({
      name: nombre.trim(),
      description: descripcion.trim()
    })
  });

  const result = await response.json();
  if (result.success) {
    await loadProjects();
    showNotification('Project created successfully', 'success');
  }
};
```

```json Request Body
{
  "name": "Legacy Migration",
  "description": "Migrate legacy system to new platform"
}
```

```json Response
{
  "success": true,
  "message": "Project created successfully",
  "data": {
    "id": 5,
    "name": "Legacy Migration",
    "description": "Migrate legacy system to new platform",
    "usercreate": 1,
    "creationDate": "2026-03-04T10:00:00",
    "active": true
  }
}
```
</CodeGroup>

<Note>
The `usercreate` field is automatically set from the authenticated user's JWT token.
</Note>

### Delete Project

Soft deletes a project by setting the `active` flag to false.

```http
DELETE /api/projects/{id}
Authorization: Bearer {token}
```

<CodeGroup>
```javascript React Frontend
const handleDelete = async () => {
  if (window.confirm('Are you sure you want to delete this project?')) {
    const response = await fetch(`/api/projects/${selectedId}`, {
      method: 'DELETE',
      headers: {
        'Authorization': `Bearer ${token}`
      }
    });

    const result = await response.json();
    if (result.success) {
      await loadProjects();
      showNotification('Project deleted successfully', 'success');
    }
  }
};
```

```json Response
{
  "success": true,
  "message": "Project deleted successfully",
  "data": null
}
```
</CodeGroup>

<Warning>
Deleting a project does not delete associated tasks. Consider reassigning tasks before deleting a project.
</Warning>


## Project-Task Relationship

Projects are linked to tasks through the `projectId` field. You can:

1. **View tasks by project** using the `/api/tasks/report/by-project` endpoint
2. **Filter tasks** by selecting a project in the Task Manager
3. **Track project progress** by viewing task statistics per project

### Example: Getting Tasks for a Project

```javascript
const getTasksForProject = async (projectId) => {
  const response = await fetch('/api/tasks', {
    headers: {
      'Authorization': `Bearer ${token}`
    }
  });
  
  const result = await response.json();
  if (result.success) {
    const projectTasks = result.data.filter(task => task.projectId === projectId);
    console.log(`Found ${projectTasks.length} tasks for project ${projectId}`);
  }
};
```

## Best Practices

<Tabs>
  <Tab title="Naming">
    Use clear, descriptive project names that reflect the work scope:
    - ✅ "Legacy System Migration"
    - ✅ "Q1 2026 API Refactoring"
    - ❌ "Project 1"
    - ❌ "Stuff"
  </Tab>
  
  <Tab title="Organization">
    - Create projects that align with your team structure or delivery milestones
    - Use the description field to provide context and goals
    - Archive (soft delete) completed projects rather than keeping them active
  </Tab>
  
  <Tab title="Task Association">
    - Always assign tasks to an appropriate project
    - Consider creating a "General" or "Miscellaneous" project for ad-hoc tasks
    - Review project task counts periodically using the reporting endpoints
  </Tab>
</Tabs>
