## Overview

The reporting system provides comprehensive analytics and insights into your tasks, projects, and team performance. Generate detailed reports filtered by state, project, or user assignment.

## Task Statistics

Get a quick overview of all task metrics with the statistics endpoint.

### GET /api/tasks/statistics

Returns aggregated statistics for all active tasks in the system.

<CodeGroup>
```bash cURL
curl -X GET http://localhost:5000/api/tasks/statistics \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```javascript JavaScript
const response = await fetch('/api/tasks/statistics', {
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  }
});
const data = await response.json();
```
</CodeGroup>

### Response Structure

<ResponseField name="success" type="boolean" required>
  Indicates if the request was successful
</ResponseField>

<ResponseField name="message" type="string" required>
  Status message
</ResponseField>

<ResponseField name="data" type="object" required>
  Statistics object containing task metrics
  
  <ResponseField name="total" type="integer">
    Total number of active tasks
  </ResponseField>
  
  <ResponseField name="completed" type="integer">
    Number of tasks with completed state (StateId = 3)
  </ResponseField>
  
  <ResponseField name="pending" type="integer">
    Number of tasks with pending state (StateId = 1)
  </ResponseField>
  
  <ResponseField name="highPriority" type="integer">
    Number of tasks with high priority (PriorityId = 3)
  </ResponseField>
  
  <ResponseField name="overdue" type="integer">
    Number of incomplete tasks past their expiration date
  </ResponseField>
</ResponseField>

### Example Response

```json
{
  "success": true,
  "message": "Statistics retrieved successfully",
  "data": {
    "total": 45,
    "completed": 12,
    "pending": 18,
    "highPriority": 8,
    "overdue": 5
  }
}
```

<Info>
  Statistics are calculated in real-time from active tasks only. Deleted (inactive) tasks are excluded from all metrics.
</Info>

## By-State Report

Analyze task distribution across different workflow states.

### GET /api/tasks/report/by-state

Groups tasks by their current state and returns the count for each state.

<CodeGroup>
```bash cURL

  -H "Authorization: Bearer YOUR_TOKEN"
```

</CodeGroup>

## Report Export

The frontend ReportManager component provides CSV export functionality for all reports.

### CSV Export Format

Exports all three report types in a single CSV file:

```csv
REPORT: TASKS
Pending: 18 tasks
In Progress: 15 tasks
Completed: 12 tasks

REPORT: PROJECTS
Legacy System Migration: 25 tasks
Mobile App Development: 12 tasks
Infrastructure Upgrade: 8 tasks

REPORT: USERS
john.doe: 15 assigned tasks
jane.smith: 12 assigned tasks
bob.wilson: 8 assigned tasks
```

### Frontend Implementation

The report manager automatically loads all three report types in parallel:

```javascript
const [tasksRes, projectsRes, usersRes] = await Promise.all([
  apiRequest('/tasks/report/by-state'),
  apiRequest('/tasks/report/by-project'),
  apiRequest('/tasks/report/by-user')
]);
```

<Info>
  Parallel loading improves performance by fetching all reports simultaneously rather than sequentially.
</Info>

## Implementation Details

### Backend Logic

Reports are generated using LINQ queries with joins to related tables:

**By-State Query** (TaskService.cs:427-460)
```csharp
var report = await (from t in _context.TblTasks
                   join s in _context.TblStates on t.StateId equals s.Id
                   where t.Active == true
                   group t by new { s.Id, s.Name } into g
                   orderby g.Key.Name
                   select new {
                       Name = g.Key.Name,
                       Count = g.Count()
                   }).ToListAsync();
```

**By-Project Query** (TaskService.cs:462-493)
```csharp
var report = await (from t in _context.TblTasks
                   join p in _context.TblProjects on t.ProjectId equals p.Id
                   where t.Active == true
                   group t by new { p.Id, p.Name } into g
                   orderby g.Key.Name
                   select new {
                       Name = g.Key.Name,
                       Count = g.Count()
                   }).ToListAsync();
```

**By-User Query** (TaskService.cs:495-526)
```csharp
var report = await (from t in _context.TblTasks
                   join u in _context.TblUsers on t.AsignedId equals u.Id
                   where t.Active == true
                   group t by new { u.Id, u.Username } into g
                   orderby g.Key.Username
                   select new {
                       Name = g.Key.Username,
                       Count = g.Count()
                   }).ToListAsync();
```
