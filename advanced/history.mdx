## Overview

The history system provides a comprehensive audit trail of all task changes. Every action performed on a task is logged with a timestamp, action type, and detailed description.

<Info>
  History records are never truly deleted - they use soft deletion (Active flag) to maintain audit trail integrity.
</Info>

## Get Task History

Retrieve all history records for a specific task.

### GET /api/history/task/{taskId}

Returns all active history records associated with a task, ordered by most recent first.

<CodeGroup>
```bash cURL
curl -X GET https://localhost:7067/api/history/task/123 \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```javascript JavaScript
const taskId = 123;
const response = await fetch(`/api/history/task/${taskId}`, {
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  }
});
const history = await response.json();
```

```csharp C#
var history = await _context.TblHistory
    .Where(h => h.TaskId == taskId && h.Active)
    .OrderByDescending(h => h.CreationDate)
    .Select(h => new HistoryDto
    {
        Id = h.Id,
        TaskId = h.TaskId,
        Action = h.Action,
        Description = h.Description,
        CreationDate = h.CreationDate,
        Active = h.Active
    })
    .ToListAsync();
```
</CodeGroup>

### Response Format

<ResponseField name="success" type="boolean" required>
  Indicates if the request was successful
</ResponseField>

<ResponseField name="data" type="array" required>
  Array of history records
  
  <ResponseField name="id" type="integer">
    Unique identifier for the history record
  </ResponseField>
  
  <ResponseField name="taskId" type="integer">
    ID of the associated task
  </ResponseField>
  
  <ResponseField name="action" type="string">
    Type of action performed (CREATED, UPDATED, STATE_CHANGED, etc.)
  </ResponseField>
  
  <ResponseField name="description" type="string">
    Detailed description of what changed
  </ResponseField>
  
  <ResponseField name="creationDate" type="datetime">
    When the action was performed
  </ResponseField>
  
  <ResponseField name="active" type="boolean">
    Whether the record is active (soft deletion flag)
  </ResponseField>
</ResponseField>

## Get All History

Retrieve the complete system-wide activity log.

### GET /api/history/all

Returns all active history records across all tasks, ordered by most recent first.

<CodeGroup>
```bash cURL
curl -X GET https://localhost:7067/api/history/all \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```javascript JavaScript
const response = await fetch('/api/history/all', {
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  }
});
const allHistory = await response.json();
```

```csharp C#
return await _context.TblHistory
    .Where(h => h.Active)
    .OrderByDescending(h => h.CreationDate)
    .Select(h => new HistoryDto
    {
        Id = h.Id,
        TaskId = h.TaskId,
        Action = h.Action,
        Description = h.Description,
        CreationDate = h.CreationDate,
```

<Warning>
  This endpoint can return a large amount of data. Consider implementing pagination for production use.
</Warning>

## Create History Record

Manually create a history entry (used for custom events or comments).

### POST /api/history

Creates a new history record for a task.

<CodeGroup>
```bash cURL
curl -X POST https://localhost:7067/api/history \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "taskId": 123,
    "action": "COMMENTED",
    "description": "User added a comment: This task needs review"
  }'
```

```javascript JavaScript
const response = await fetch('/api/history', {
  method: 'POST',
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    taskId: 123,
    action: 'COMMENTED',
    description: 'User added a comment: This task needs review'
  })
});
const result = await response.json();
```
</CodeGroup>

### Request Body

<ResponseField name="taskId" type="integer" required>
  ID of the task to log history for
</ResponseField>

<ResponseField name="action" type="string" required>
  Type of action (CREATED, UPDATED, COMMENTED, etc.)
</ResponseField>

<ResponseField name="description" type="string" required>
  Detailed description of the action
</ResponseField>

### Response

```json
{
  "success": true,
  "data": {
    "id": 457,
    "taskId": 123,
    "action": "COMMENTED",
    "description": "User added a comment: This task needs review",
    "creationDate": "2026-03-04T15:45:00",
    "active": true
  }
}
```

## Delete History Record

Soft delete a history record (sets Active flag to false).

### DELETE /api/history/{historyId}

Marks a history record as inactive without actually deleting it from the database.

<CodeGroup>
```bash cURL
curl -X DELETE https://localhost:7067/api/history/457 \
  -H "Authorization: Bearer YOUR_TOKEN"
```

```javascript JavaScript
const historyId = 457;
const response = await fetch(`/api/history/${historyId}`, {
  method: 'DELETE',
  headers: {
    'Authorization': `Bearer ${token}`,
    'Content-Type': 'application/json'
  }
});
const result = await response.json();
```

```csharp C#
var history = await _context.TblHistory.FindAsync(historyId);
if (history == null)
    return false;

history.Active = false;
await _context.SaveChangesAsync();
return true;
```
</CodeGroup>

### Response

```json
{
  "success": true,
  "message": "History deleted successfully"
}
```

<Note>
  Deleted history records remain in the database with `Active = false` to maintain audit trail integrity.
</Note>

## Best Practices

<Steps>
  <Step title="Descriptive Messages">
    Write clear, actionable history descriptions that include who made the change and what changed.
    
    Good: "Priority changed from 'Low' to 'High' by john.doe"
    
    Bad: "Updated"
  </Step>
  
  <Step title="Consistent Action Types">
    Use standardized action type strings to make filtering and reporting easier.
  </Step>
  
  <Step title="Never Hard Delete">
    Always use soft deletion (Active flag) to maintain complete audit trail.
  </Step>
  
  <Step title="Include Context">
    When logging changes, include before/after values to make the history meaningful.
  </Step>
  
  <Step title="Timestamp Accuracy">
    Use server-side timestamps (DateTime.Now) rather than client-side to ensure consistency.
  </Step>
</Steps>
