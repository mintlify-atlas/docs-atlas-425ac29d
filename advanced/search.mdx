## Overview

The search functionality allows you to find tasks quickly using multiple filters. Combine text search with state, priority, and project filters to narrow down results.

<Info>
  Search is performed client-side after fetching all tasks, allowing for instant filtering without additional API calls.
</Info>

### Step 3: Apply Filters

Filters are applied client-side in the following order:

<Steps>
  <Step title="Text Filter">

    ```javascript

        task.title?.toLowerCase().includes(searchLower) ||
        task.description?.toLowerCase().includes(searchLower)
      );
    }
    ```
  </Step>
  
  <Step title="State Filter">
    Match exact state ID
    
    ```javascript
    if (selectedState) {
      filteredTasks = filteredTasks.filter(task => 
        task.stateId === parseInt(selectedState)
      );
    }
    ```
  </Step>
  
  <Step title="Priority Filter">
    Match exact priority ID
    
    ```javascript
    if (selectedPriority) {
      filteredTasks = filteredTasks.filter(task => 
        task.priorityId === parseInt(selectedPriority)
      );
    }
    ```
  </Step>
  
  <Step title="Project Filter">
    Match exact project ID
    
    ```javascript

      filteredTasks = filteredTasks.filter(task => 
        task.projectId === parseInt(selectedProject)

    }
    ```

</Steps>

## Frontend Implementation

### Search Component

The SearchManager component (SearchManager.jsx:54-94) implements the search logic:

```javascript
const handleSearch = async () => {
  setLoading(true);
  setHasSearched(true);
  try {
    const response = await apiRequest('/tasks');
    
    if (response.success) {
      let filteredTasks = response.data;

      // Filter by text
      if (searchText.trim()) {
        const searchLower = searchText.toLowerCase();
        filteredTasks = filteredTasks.filter(task =>
          task.title?.toLowerCase().includes(searchLower) ||
          task.description?.toLowerCase().includes(searchLower)
        );
      }

      // Filter by state
      if (selectedState) {
        filteredTasks = filteredTasks.filter(task => 
          task.stateId === parseInt(selectedState)
        );
      }

      // Filter by priority
      if (selectedPriority) {
        filteredTasks = filteredTasks.filter(task => 
          task.priorityId === parseInt(selectedPriority)
        );
      }

      // Filter by project
      if (selectedProject) {
        filteredTasks = filteredTasks.filter(task => 
          task.projectId === parseInt(selectedProject)
        );
      }

      setTasks(filteredTasks);
    }
  } catch (error) {
    console.error('Error searching tasks:', error);
  } finally {
    setLoading(false);
  }
};
```

### Enter Key Support

The search input supports pressing Enter to trigger search:

```javascript
<input
  type="text"
  value={searchText}
  onChange={(e) => setSearchText(e.target.value)}
  onKeyPress={(e) => e.key === 'Enter' && handleSearch()}
  placeholder="Search in title or description..."
/>
```

## Search Results Display

Results are displayed in a table format with visual indicators:

### State Colors

Tasks are color-coded based on their state:

- **Pending** (StateId 1): Orange
- **In Progress** (StateId 2): Blue
- **Completed** (StateId 3): Green
- **Other states**: Gray

```javascript
const getStateColor = (stateId) => {
  const state = states.find(s => s.id === stateId);
  if (!state) return 'bg-gray-50 hover:bg-gray-100';
  
  const nameUpper = state.name?.toUpperCase() || '';
  if (nameUpper.includes('PENDING')) return 'bg-orange-50 hover:bg-orange-100';
  if (nameUpper.includes('IN PROGRESS')) return 'bg-blue-50 hover:bg-blue-100';
  if (nameUpper.includes('COMPLETED')) return 'bg-green-50 hover:bg-green-100';
  return 'bg-gray-50 hover:bg-gray-100';
};
```

### Priority Colors

Priorities are displayed with badges:

- **High**: Red badge
- **Medium**: Amber badge
- **Low**: Green badge

```javascript
const getPriorityColor = (priorityId) => {
  const priority = priorities.find(p => p.id === priorityId);
  if (!priority) return 'bg-red-50 text-red-700 border-red-200';
  
  const nameUpper = priority.name?.toUpperCase() || '';
  if (nameUpper.includes('HIGH')) return 'bg-red-50 text-red-700 border-red-200';
  if (nameUpper.includes('MEDIUM')) return 'bg-amber-50 text-amber-700 border-amber-200';
  if (nameUpper.includes('LOW')) return 'bg-green-50 text-green-700 border-green-200';
  return 'bg-gray-50 text-gray-700 border-gray-200';
};
```

## Performance Considerations

<CardGroup cols={2}>
  <Card title="Parallel Loading" icon="bolt">
    Filter options (states, priorities, projects) are loaded in parallel using `Promise.all()` for faster initialization.
  </Card>
  <Card title="Client-Side Filtering" icon="filter">
    All filtering happens client-side after initial data fetch, providing instant results without API latency.
  </Card>
  <Card title="Single API Call" icon="server">
    Only one API call to `/api/tasks` is made per search, reducing server load.
  </Card>
  <Card title="Responsive UI" icon="mobile">
    The search interface is fully responsive and works on all device sizes.
  </Card>
</CardGroup>

<Warning>
  For large datasets (thousands of tasks), consider implementing server-side pagination and filtering to improve performance.
</Warning>

## Best Practices

<Steps>
  <Step title="Combine Multiple Filters">
    Use multiple filters together for precise results. For example, search for "database" + High priority + In Progress state.
  </Step>
  
  <Step title="Use Text Search Wisely">
    Text search checks both title and description fields, so be specific to avoid too many results.
  </Step>
  
  <Step title="Clear Filters Between Searches">
    Reset filters when starting a new search to avoid confusion from previous criteria.
  </Step>
  
  <Step title="Bookmark Common Searches">
    Consider implementing saved search presets for frequently used filter combinations.
  </Step>
</Steps>
